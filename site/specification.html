Framecurve specification

A framecurve file is a file with ASCII text data and it must have a double extension of .framecurve.txt

Inside the framecurve file there is a number of records, each line is a record. Records must be delimited by the CRLF line delimiters to facilitate editing the file on Windows.

Each record can be either a frame correlation record or a comment. A comment is a line starting with a hashmark (#).

# This is a comment

or a frame correlation record. A frame correlation record represents a record of two fields, having the format of

[destination_frame_integer][TAB][source_frame_float][CRLF]

The destination frame must be an integer value specifying the destination frame in the edit. The source frame must be a floating point decimal value representing the frame that will be shown at this frame.

The destination frame values must be consecutive (the destination frame number has to increase from record to record). This is not valid:

12	34.5
8	20.5
42	20.5

Destination frame values must be unique.

This is not valid: 
12	34.5
12	20.5
42	20.5

The first line of a framecurve file should include a comment line noting that this is a framecurve file, and it's recommended to include the interpretation of the record columngs as well. This is a recommended header.
